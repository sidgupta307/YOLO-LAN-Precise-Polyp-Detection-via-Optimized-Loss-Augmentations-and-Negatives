# Code Repository for IEEE Access Submission  
**YOLO-LAN: Polyp Detection with M2IoU Loss, Augmentation, and Cross-Domain Evaluation**

---

## 1. Purpose of This Repository

This repository contains the **complete experimental code** used in the research paper submitted to **IEEE Access**.  
The code is provided to support **reproducibility, transparency, and independent verification** of the results reported in the manuscript.

All experiments, ablation studies, and analyses described in the paper are implemented in the provided Jupyter notebooks.

---

## 2. Datasets Used

The following publicly available datasets are used:

- **Kvasir-seg** (primary training dataset)
- **BKAI-IGH NeoPolyp** (second training dataset)
- **CVC-ClinicDB**
- **CVC-ColonDB**
- **ETIS-Larib**

All dataset usage follows the original licenses and terms provided by the dataset creators.

---

## 3. Dataset Splitting Protocol

For **both Kvasir-seg and BKAI-IGH NeoPolyp**, the datasets were split as follows:

- **70% Training**
- **20% Validation**
- **10% Testing**

The test and validation sets were kept **strictly unseen** during training and ablation experiments.

---

## 4. Models and Variants

The following object detection models were used:

- **YOLOv8**: nano, small, medium, large variants
- **YOLOv12**: nano, small, medium, large variants

All models were trained using **200 epochs** under identical experimental settings for fair comparison.

---

## 5. Loss Function Modification

- The default **CIoU loss** in YOLO was replaced with the proposed **M2IoU loss**.
- Comparative experiments were conducted using:
  - CIoU loss
  - M2IoU loss

This modification is central to the proposed YOLO-LAN framework.

---

## 6. Ablation Study Design

All ablation studies were conducted using **700 training images from the Kvasir-seg dataset and BKAI-IGH Neopolyp dataset** unless otherwise stated.

### 6.1 Baseline Experiments
1. YOLOv8 and YOLOv12 trained with **CIoU loss**
2. YOLOv8 and YOLOv12 trained with **M2IoU loss**

(No augmentation, no negative samples)

---

### 6.2 Data Augmentation Experiments

The following augmentation strategies were applied:

- Horizontal flip  
- Vertical flip  
- Horizontal + vertical flip  
- Rotation (45°)  
- Horizontal flip + 45° rotation  
- Vertical flip + 45° rotation  
- Horizontal + vertical flip + 45° rotation  
- Gaussian blur  
- Gaussian blur + 45° rotation  

After augmentation:
- Training set size increased from **700 to 7000 images**
- Models trained: YOLOv8 and YOLOv12 with **M2IoU loss**

---

### 6.3 Negative Sample Inclusion

To reduce false positives, negative (non-polyp) samples were added from PolypGen2020 and Kvasir datasets:

- **10% negative samples**
  - Training images: **8400**
- **20% negative samples**
  - Training images: **9880**

All models were trained using **M2IoU loss**.

---

### 6.4 CLAHE Preprocessing

Finally, **CLAHE (Contrast Limited Adaptive Histogram Equalization)** preprocessing was applied to:

- Augmented data
- 20% negative sample configuration

Training was repeated using:
- YOLOv8 and YOLOv12
- All model size variants
- M2IoU loss

---

## 7. Evaluation Metrics

For all experiments, the following metrics were computed:

- Precision  
- Recall  
- F1-score  
- mAP@50  
- mAP@50:95  

All reported values in the manuscript correspond directly to outputs generated by these notebooks.

---

## 8. Model Saving and Test Evaluation

- Trained models for all experimental configurations were saved
- Performance analysis was conducted across:
  - YOLOv8 and YOLOv12
  - Nano, small, medium, large variants
- Final evaluation was performed using the **10% test split**

---

## 9. Cross-Domain Generalization Evaluation

To evaluate robustness and generalization:

- The **best-performing model** was selected:
  - YOLOv8-small
  - Trained on Kvasir-seg
  - Augmentation + 20% negative samples + CLAHE
  - M2IoU loss

This model was evaluated on:
- BKAI-IGH NeoPolyp (test split)
- CVC-ClinicDB
- CVC-ColonDB
- ETIS-Larib

The corresponding inference and evaluation code is included.

---

## 10. Polyp Size–Wise Performance Analysis

To analyze localization performance across polyp sizes:

- Kvasir-seg test images were divided into three groups based on bounding box area relative to image size:
  - **Small polyps:** < 5%
  - **Medium polyps:** 5–15%
  - **Large polyps:** > 15%

For each image:
- Predicted bounding boxes are shown in **red**
- Ground truth bounding boxes are shown in **green**

This analysis enables identification of:
- Over-extended bounding boxes
- Under-extended bounding boxes
- Missed detections
- Localization inaccuracies across size categories
